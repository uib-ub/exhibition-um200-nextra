# UI components

This is a guide for the UI components that can be used to create content for the exhibition website. Each component has a basic usage example and a more complex example.

## Components and Markdown

You can use the following components and Markdown to create content for the exhibition website. Be aware that you can use _markdown_ inside the components. We can even use HTML tags inside the components, like `<u>underline</u>` and `<cite>cite</cite>`. Most likly you are fine by just using _markdown_.

## Work

`<Work />` is an IIIF viewer that can be used to display digital IIIF manifests. It is built with [Clover-IIIF](https://github.com/samvera-labs/clover-iiif).

### Basic usage

`Work` Can be use in several ways. The simplest way is to just pass the IIIF manifest URL to the `iiifContent` prop and have the `Work` component be "self closing". This is useful if you want to display a single IIIF manifest without any additional content. `<Work iiifContent="some url" />`

<Tabs items={['Preview', 'Code']}>
  <Tabs.Tab>
    <Work iiifContent="https://api.ub.uib.no/items/ubb-diplom-0034?as=iiif" />
  </Tabs.Tab>
  <Tabs.Tab>
    ```tsx copy
    <Work iiifContent="https://api.ub.uib.no/items/ubb-diplom-0034?as=iiif" />
    ```
  </Tabs.Tab>
</Tabs>

### With description

`Work` can also be used with a description. The description is displayed in a card below the IIIF viewer. Now the ´Work´ component needs to be closed with a `</Work>` tag as we want to add content inside the `Work` component.

<Tabs items={['Preview', 'Code']}>
  <Tabs.Tab>
    <Work iiifContent="https://api.ub.uib.no/items/ubb-bros-00464?as=iiif">
      <Work.Content>
        <Work.Description>
          This is an example of how you can display a digital object in your exhibition. 
          Replace this text with your own description of the object you want to show.
        </Work.Description>
      </Work.Content>
    </Work>
  </Tabs.Tab>
  <Tabs.Tab>

### Content state

To link directly to a page or _canvas_ in IIIF speak, we need to create a _content state_. A content state is a way of sharing a "view" of a IIIF manifest.

The easiest way to create a content state is to use the [Clover IIIF Viewer website](https://github.com/samvera-labs/clover-iiif) and paste in the URL of the IIIF manifest you want to start with, for example https://api.ub.uib.no/items/ubb-diplom-0034?as=iiif. Navigate to the page you want to share and scroll down to the three boxes below the IIIF viewer. The first box is where you add your own text about the _content state_, e.g. your _annotation_ about the page. Your an even zoom in and the annotation will be about that region of the page! Cool, indeed.

When you are happy, copy the Base64 text from the second box and use that in the `Work` components `iiifContent` prop. 

The third boxs shows you the actual content of the Base64 text, it is a JSON object! Pretty dope.

## Figure

`<Figure />` is a component that can be used to display an image with a description and a link. **NB!** You _must_ add a `alt` text that describes the image. Describe the image in a way that is accessible to everyone, including people who use screen readers. "Man smoking a pipe wearing a striking jacket" is a good alt text. "Ole Olsen, director of the National Archives during the 1930s, was a very important person" is not. Thats a caption, not an `alt` text!

### Basic usage

Using an image from the `public/images` folder, but you can link to an image. But tell the developer as some changes needs to be done for it to show.

<Tabs items={['Preview', 'Code']}>
  <Tabs.Tab>
    <Figure image="/images/arkeologi/B2978_Myklebustmannen.jpg" alt="Myklebustmannen" width={1000} height={1000} />
  </Tabs.Tab>
  <Tabs.Tab>
    ```tsx copy
    <Figure image="/images/arkeologi/B2978_Myklebustmannen.jpg" alt="Myklebustmannen" width={1000} height={1000} />
    ```
  </Tabs.Tab>
</Tabs>
